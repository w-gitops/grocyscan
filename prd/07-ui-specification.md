## 7.3 Component Standards

### 7.3.1 Detail Popup (70% Width)

All detail views use a 70% viewport width popup dialog.

```python
from nicegui import ui

class DetailPopup:
    @staticmethod
    def show(title: str, content_builder, on_confirm=None, on_cancel=None):
        with ui.dialog() as dialog, ui.card().classes('w-[70vw] max-h-[90vh] overflow-auto'):
            # Header
            with ui.row().classes('w-full justify-between items-center'):
                ui.label(title).classes('text-xl font-bold')
                ui.button(icon='close', on_click=dialog.close).props('flat round')
            
            ui.separator()
            
            # Content area
            content_builder()
            
            # Footer with action buttons
            if on_confirm or on_cancel:
                ui.separator()
                with ui.row().classes('w-full justify-end gap-4'):
                    if on_cancel:
                        ui.button('Cancel', on_click=lambda: [on_cancel(), dialog.close()])
                    if on_confirm:
                        ui.button('Confirm', on_click=lambda: [on_confirm(), dialog.close()])
        
        dialog.open()
```

class TouchDatePicker:

## 7.4 Responsive Design Guidelines

| Breakpoint | Width | Layout Adjustments |
|------------|-------|-------------------|
| **Mobile** | <640px | Single column, bottom navigation, full-width buttons |
| **Tablet** | 640-1024px | Two columns where appropriate, side navigation |
| **Desktop** | >1024px | Full layout with sidebar, multi-column grids |

## 7.5 Touch Optimization Standards

- **Minimum tap target**: 48x48 pixels
- **Button padding**: `py-4 px-6` minimum (16px vertical, 24px horizontal)
- **Spacing between interactive elements**: 8px minimum
- **Swipe gestures**: Support where appropriate (date picker, list items)
